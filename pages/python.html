Maze Game: Python

About the Project

For my project, I was assigned to develop a running program that implements a straightforward two-player game. The coding had to be done in Python without the aid of a pre-made game engine, although Pygame is an exception which I chose to use. I aimed to develop a Maze Game that generates a new maze with each playthrough, incorporating a timer and a scoring system based on the outcome. Crafting the maze generator proved to be a complex task, demanding significant problem-solving skills. To ensure fairness, I designed the game so that players begin at opposite ends of the maze, aiming to reach each other's starting point. This setup proved most effective and visually appealing compared to a central starting point. I encountered challenges with the countdown display, where numbers overlapped, and resolved this by overlaying the countdown on a distinct shape for clarity.

Game Demo

To prepare players before diving in, I introduced a splash page, setting the game commencement at 40 seconds. A completion by any player triggers an announcement of the winner and auto-restarts the game, with a new maze and the option to manually restart by pressing 'r', adding a dynamic play-as-you-wish element. Points accrue with each win. Should time expire, a game over alert appears with a restart prompt, but no points are awarded.
